## 1. 전투 시작 시 세팅

### 1-1. 전투 시작 거리 결정

두 기체(플레이어 함선, 적 함선) 중 **더 긴 레이더 탐지 범위를 가진 쪽의 레이더 한계 거리**에서 전투가 시작된다.

> **예시**  
> 내 함선 (공격 의도 없음) 레이더 거리: **1300**  
> 적 함선 (공격 의도 있음) 레이더 거리: **700**  
> → 전투 시작 거리: **1300**

- 레이더 장비는 `electronic_warfare` 카테고리에 속하며, 장비별로 탐지 범위가 다를 수 있다.
- 공격 의도 여부와 관계없이, 양 측의 레이더 거리 중 **최대값**을 전투 시작 거리로 채택한다.

### 1-2. 턴(프레임) 순서 결정

한 턴은 **'프레임'** 이라는 단위를 사용한다.  
플레이어와 적 함선을 비교하여, **함선 급이 낮은(소형인) 순서대로 행동을 먼저 시작**한다.

#### 함선 급별 행동 순서 (빠른 순)

| 순서 | 함선 급 |
|------|---------|
| 1 | 소형 기체 |
| 2 | 중형 기체 |
| 3 | 상업용 기체 |
| 4 | 구축함 (Destroyer) |
| 5 | 순양함 (Cruiser) |
| 6 | 중순양함 |
| 7 | 경전함 |
| 8 | 순양전함 (Battlecruiser) |
| 9 | 전함 (Battleship) |
| 10 | 거대전함 |

> 소형 기체가 가장 먼저 턴을 시작하고, 거대전함이 가장 늦게 행동한다.

#### 현재 구현된 함선과의 매핑

| 게임 데이터 ID | 한글명 | 함선 급 분류 |
|---------------|--------|-------------|
| `corvette` | 초계함 | 소형 기체 |
| `frigate` | 호위함 | 중형 기체 |
| `cargo_ship` | 화물선 | 상업용 기체 |
| `destroyer` | 구축함 | 구축함 |
| `cruiser` | 순양함 | 순양함 |
| `drone_carrier` | 드론항모 | 중순양함 |
| `assault_ship` | 상륙강습함 | 경전함 |
| `battlecruiser` | 순양전함 | 순양전함 |
| `battleship` | 전함 | 전함 |
| `carrier` | 항모 | 거대전함 |

### 1-3. 동일 급 함선의 순서 결정

같은 급의 함선끼리 교전할 경우, **엔진(engine)의 총 출력이 높은 쪽이 먼저 행동**한다.

- 엔진 장비는 `engine` 카테고리에 해당하며, 장착된 엔진 장비의 총 출력 합산으로 비교한다.
- 현재 엔진 장비 예시: `booster_engine` (부스터 엔진), `warp_drive` (워프 드라이브) 등

### 1-4. 대기열(Queue) 초기화

전투 시작 시, **모든 활성화된 장비에 따라 '대기열(Queue)' 카드 덱에 카드가 채워진다.**

- 각 장비의 `providedCards` 배열에 정의된 카드 ID와 수량만큼 대기열에 카드가 추가된다.
- 장비가 피격 등의 이유로 비활성화되면, 해당 장비가 제공하는 카드는 대기열에 포함되지 않는다.

```
장비 배치 상태 (예시)
┌──────────────────┐     ┌──────────────────┐
│  레이저 캐논      │     │  실드 제너레이터   │
│  → 레이저 사격 ×2 │     │  → 에너지 실드 ×2 │
└──────────────────┘     └──────────────────┘
         │                        │
         └────────┬───────────────┘
                  ▼
        ┌─────────────────┐
        │    대기열 (Queue) │
        │  레이저 사격 ×2  │
        │  에너지 실드 ×2  │
        │  ... (기타 카드)  │
        └─────────────────┘
```

## 2. 턴(프레임) 진행

각 프레임은 아래 순서대로 진행된다.

### 2-1. 손패(핸드) 드로우

함선 컴퓨터의 성능에 따라 **'대기열(Queue)'에서 손패(핸드)로 사용 가능한 카드를 가져온다.**

- 컴퓨터 성능이 높을수록 한 턴에 더 많은 카드를 손패에 가져올 수 있다.
- 대기열에서 가져온 카드만 해당 턴에 사용할 수 있다.

### 2-2. EP(Energy Point) 수급

함선의 **제네레이터, 핵융합로 등 동력원**에 따라 프레임당 사용 가능한 **EP(Energy Point)** 를 받는다.

- EP는 카드를 플레이하기 위한 에너지 자원이다.
- 동력 관련 장비(generator 카테고리)의 성능에 비례하여 EP가 결정된다.
- 기존 Planning.md의 AP(Action Point) 시스템과 연계하여 운용된다.

> **참고**: 현재 `types.ts`의 `CardDefinition`에서는 `epCost`(에너지 포인트 비용)를 사용한다.  
> EP 시스템이 도입되면 AP와 EP 두 자원을 복합적으로 관리하게 된다.

### 2-3. 패시브 효과 적용

함선의 **장비 패시브 효과가 턴 시작 시 자동으로 적용**된다.

적용되는 효과 예시:

| 패시브 효과 | 설명 |
|------------|------|
| 장갑 재건 | 매 턴 시작 시 일정량의 장갑(방어력)을 회복 |
| 쉴드 충전 | 매 턴 시작 시 쉴드(에너지 보호막) 충전 |
| 재장전 | 특정 무기 카드의 쿨다운/재사용 대기 감소 |
| HP 회복 | 생명유지장치 등의 패시브로 매 턴 HP 회복 |

현재 구현된 패시브 예시:
- `life_support_passive`: 매 턴 HP +2 회복
- 각 함선별 고유 패시브 (`ShipPassive` — `on_turn_start` 트리거 등)

### 2-4. 카드 플레이

**탄약, 에너지(EP), 행동력(AP) 등을 고려하여 핸드에 들어온 카드를 플레이한다.**

```
[핸드 카드 확인]
    │
    ├─ 카드 사용 조건 확인
    │   ├─ EP(에너지) 충분한가?
    │   ├─ AP(행동력) 충분한가?
    │   └─ 탄약/특수 자원이 있는가?
    │
    ├─ 조건 충족 → 카드 효과 발동
    │   ├─ attack: 적에게 피해
    │   ├─ defense: 쉴드/장갑 강화
    │   ├─ skill: 회피/버프/유틸리티
    │   └─ passive: 자동 적용 (AP 0)
    │
    └─ 남은 EP/AP 확인 → 추가 카드 사용 또는 턴 종료
```

### 2-5. 카드 소진 및 덱 리로드

행동에 사용된 카드들은 **'소진(Exhaust)' 카드 덱**에 들어간다.

- 소진된 카드는 **모든 '대기열' 안의 카드가 플레이될 때까지**(덱 리로드 시점) 재등장하지 않는다.
- 덱 리로드가 발생하면, 소진 덱의 카드들이 다시 대기열로 돌아간다.
- **특수 효과**로 소진 덱에서 직접 카드를 가져오는 경우는 예외이다.

```
[카드 흐름]

  대기열 (Queue)  ──드로우──▶  손패 (Hand)  ──사용──▶  소진 (Exhaust)
       ▲                                                    │
       │                                                    │
       └──────────── 덱 리로드 (대기열 소진 시) ◀────────────┘
```

### 2-6. 제외(Exclude) 카드

특수 효과 등으로 **'제외(Exclude)' 카드 덱**에 들어간 카드는, 덱 리로드가 일어나도 **대기열 카드 덱에 돌아가지 않는다.**

- 제외된 카드는 해당 전투가 끝날 때까지 영구적으로 사용 불가능하다.
- 저주 장비, 특정 적의 디버프 등에 의해 카드가 제외될 수 있다.

```
[카드 덱 구조 전체]

  ┌────────────┐
  │  대기열     │ ◀─── 장비로부터 초기 채워짐
  │  (Queue)   │ ◀─── 덱 리로드 시 소진에서 복귀
  └─────┬──────┘
        │ 드로우
        ▼
  ┌────────────┐
  │   손패      │
  │  (Hand)    │
  └─────┬──────┘
        │ 사용
        ▼
  ┌────────────┐        ┌────────────┐
  │   소진      │        │   제외      │
  │ (Exhaust)  │        │ (Exclude)  │
  │  → 리로드   │        │  → 영구 제거 │
  └────────────┘        └────────────┘
```

## 3. 전투 처리 (피해 판정)

### 3-1. 피해 적용 우선순위

공격이 들어올 때, 피해는 아래 순서대로 적용된다:

```
[공격 피해]
    │
    ▼
┌──────────┐    고갈    ┌──────────┐    고갈    ┌────────────────┐
│  쉴드     │ ────────▶ │  장갑     │ ────────▶ │ 함선 내부 피탄   │
│ (Shield) │           │ (Armor)  │           │ (랜덤 공간 피격) │
└──────────┘           └──────────┘           └────────────────┘
```

1. **쉴드(Shield)** 가 가장 먼저 소모된다.
   - 쉴드 장비(`shield` 카테고리)에 의해 충전/강화된다.
   - 쉴드가 남아 있으면 피해를 쉴드에서 차감한다.

2. **장갑(Armor)** 이 그다음으로 소모된다.
   - 장갑 장비(`armor` 카테고리)에 의해 제공된다.
   - 쉴드가 모두 고갈된 후, 남은 피해가 장갑에서 차감된다.

3. **함선 내부 피탄** — 쉴드와 장갑이 모두 고갈된 상태에서 공격이 들어오면, **함선의 랜덤한 공간(그리드 셀)에 직접 피탄**된다.
   - 피탄 위치는 함선 그리드의 유효 셀(`validCells`) 중 장비가 배치된 곳에서 무작위로 결정된다.

### 3-2. 장비 파손 처리

**피탄된 곳에 배치된 장비에 소속된 카드는 즉시 '사용불가' 처리**된다.

- 해당 장비의 `providedCards`에 해당하는 모든 카드가 즉시 **사용불가** 상태로 전환된다.
- 사용불가 카드는 손패에 들어와도 플레이할 수 없다.
- 대기열, 소진 덱 어디에 있든 상관없이 즉시 비활성화된다.

```
[장비 피탄 예시]

  레이저 캐논 피탄!
      │
      ▼
  레이저 사격 카드 ×2 → 즉시 [사용불가] 상태
  (손패에 있어도 플레이 불가)
```

### 3-3. 게임오버 조건

**브릿지, 함장실, 선원실이 모두 피격되거나 기능정지 되면 즉시 게임오버.**

| 핵심 구역 | 역할 |
|-----------|------|
| 브릿지 (Bridge) | 함선 조종 및 지휘 중추 |
| 함장실 (Captain's Quarters) | 함장의 지휘 기능 |
| 선원실 (Crew Quarter) | 함선 운영 인원 거주 공간 |

- 세 구역은 함선 운영의 핵심이며, **하나라도 남아 있으면 전투를 계속**할 수 있다.
- 세 구역이 **모두 파괴 또는 기능정지**될 경우, 함선을 운용할 수 없으므로 **즉시 게임오버**가 된다.

> **참고**: 현재 데이터에서 `crew_quarter` 카테고리가 선원실에 해당한다.  
> 브릿지와 함장실은 향후 장비 또는 구역 데이터로 추가 구현이 필요하다.

## 전투 흐름 전체 요약

```
[전투 시작]
  │
  ├─ 1. 전투 시작 거리 = max(플레이어 레이더, 적 레이더)
  ├─ 2. 턴 순서 결정 (함선 급 → 엔진 출력)
  └─ 3. 대기열(Queue) 초기화 (활성 장비 기반)
  │
  ▼
[프레임 (턴) 반복]
  │
  ├─ 2-1. 손패 드로우 (컴퓨터 성능 기반)
  ├─ 2-2. EP 수급 (동력원 기반)
  ├─ 2-3. 패시브 효과 적용 (장갑 재건, 쉴드 충전, 재장전 등)
  ├─ 2-4. 카드 플레이 (EP/AP/탄약 소모)
  ├─ 2-5. 사용 카드 → 소진 덱 (덱 리로드까지 대기)
  └─ 2-6. 제외 카드 → 영구 제거 (리로드 불가)
  │
  ▼
[피해 판정]
  │
  ├─ 3-1. 쉴드 → 장갑 → 함선 내부 랜덤 피탄
  ├─ 3-2. 피탄 장비 카드 즉시 사용불가
  └─ 3-3. 브릿지 + 함장실 + 선원실 모두 파괴 → 게임오버
  │
  ▼
[전투 종료 조건]
  ├─ 적 함선 격파 → 승리
  ├─ 플레이어 HP 0 → 패배
  └─ 핵심 구역 전멸 → 즉시 게임오버
```
