## 2. 핵심 메커니즘

### 2-1. 우주선 그리드 인벤토리

#### 개념
- 우주선 내부는 **정사각형 셀(cell)로 이루어진 2D 그리드**로 표현된다.
- 그리드의 **형태(배치된 셀의 집합)** 자체가 우주선의 종류를 결정한다.
- 셀에 장비를 배치하는 것이 곧 **인벤토리 관리**이며, 배치된 장비만 전투에서 사용할 수 있다.

#### 그리드 규칙
- 각 셀은 세 가지 상태를 가진다:
  - `EMPTY` — 빈 공간, 장비 배치 가능
  - `OCCUPIED` — 장비가 배치된 상태
  - `BLOCKED` — 구조적으로 사용 불가 (우주선 외벽 등)
- 장비는 그리드 안에서 **회전(0°, 90°, 180°, 270°)** 가능하다.
- 장비는 **서로 겹칠 수 없다**.
- 장비를 탈착(unequip)하면 해당 카드는 덱에서 즉시 제거된다.

```
예시 — "소형 정찰선" 그리드 (5×5, 유효 셀 표시)

  . X X X .
  X X X X X
  X X X X X
  . X X X .
  . . X . .
```

---

### 2-2. 스테이지 진행 구조

Slay the Spire의 **맵 노드** 방식을 채택한다.

```
[출발]
  ├─ 전투 ── 전투 ── 상점
  ├─ 이벤트 ── 전투 ──┤
  └─ 전투 ── 이벤트──┘
              ▼
           [엘리트 전투]
              ▼
           [휴식 / 수리 기지]
              ▼
           [보스 전투]
              ▼
           [다음 구역]
```

- 각 **구역(Zone)** 은 3–5개의 노드 레이어로 구성된다.
- 총 **3개 구역** + **최종 보스** 구성을 기본으로 한다.
- 플레이어는 분기 경로 중 하나를 선택해 진행한다.

#### 노드 타입

| 노드 | 설명 |
|------|------|
| ⚔️ 전투 | 적 우주선과 카드 전투 |
| ⭐ 엘리트 전투 | 강화된 적, 높은 보상 |
| 👾 보스 전투 | 구역 마지막, 특수 패턴 |
| 🛍️ 상점 | 장비·카드 업그레이드 구매 |
| ❓ 이벤트 | 선택지 기반 랜덤 이벤트 |
| 🔧 수리 기지 | 함선 내구도 회복 & 장비 재배치 |

### 2-3. 장비 시스템 (테트리스 모양)

#### 장비 = 폴리오미노(Polyomino) 조각
- 모든 장비는 **1×1 ~ 최대 5×5 범위의 폴리오미노** 형태를 가진다.
- 예를 들어 "레이저 캐논"은 1×3 직선, "실드 제너레이터"는 L자형 3칸 등.
- 장비 배치 시 **색상**으로 종류를 구분한다.

#### 장비 등급
| 등급 | 색상 | 특징 |
|------|------|------|
| 일반(Common) | 회색 | 단순 효과, 1–2칸 |
| 희귀(Rare) | 파랑 | 복합 효과, 2–4칸 |
| 에픽(Epic) | 보라 | 강력한 시너지, 3–5칸 |
| 전설(Legendary) | 황금 | 고유 패시브 포함, 3–5칸 |

#### 장비 카테고리
각 카테고리의 장비는 전투에서 **카드**로 변환된다. (→ 2-4 참조)

| 카테고리 | 주 역할 |
|----------|---------|
| 무기(Weapon) | 공격 카드 생성 |
| 미사일(Missile) | 광역/상태이상 공격 |
| 실드(Shield) | 방어/블록 카드 |
| 엔진(Engine) | 이동/회피 카드 |
| 선원실(Crew Quarter) | 버프/회복 카드 |
| 의무실(Med Bay) | 치유/상태이상 제거 |
| 생명유지장치(Life Support) | 지속 HP 회복 패시브 |

### 2-4. 카드 덱 전투

#### 덱 구성 원리
- **함선에 설치된 장비의 수와 종류** = 덱에 들어가는 카드 목록
- 예: 레이저 캐논 1개 설치 → 덱에 "레이저 사격" 카드 2장 추가
- 장비를 제거하면 관련 카드도 덱에서 빠진다.
- 같은 종류의 장비를 여러 개 설치하면 해당 카드가 복수로 추가된다.

#### 전투 흐름
```
[전투 시작]
  └─ 덱 셔플 → 5장 드로우

[플레이어 턴]
  1. 보유 액션 포인트(AP) 확인 (기본 3 AP/턴)
  2. 핸드에서 카드 선택 → AP 소모 → 효과 발동
  3. AP가 0이 되거나 "종료" 선택 시 턴 종료

[적 턴]
  └─ 적의 의도(예: 공격 8, 방어 준비) 실행

[반복]
  └─ 어느 쪽이든 HP 0이 되면 전투 종료
```

#### 액션 포인트(AP)
- 기본 3 AP/턴
- 엔진 장비 → 최대 AP 증가 가능
- 선원실 장비 → 턴 시작 시 드로우 +1 가능

#### 카드 예시

| 카드명 | AP 비용 | 효과 | 출처 장비 |
|--------|---------|------|-----------|
| 레이저 사격 | 1 | 적에게 6 피해 | 레이저 캐논 |
| 미사일 발사 | 2 | 모든 적에게 4 피해, 화상 1 부여 | 미사일 런처 |
| 에너지 실드 | 1 | 블록 8 획득 | 실드 제너레이터 |
| 긴급 회피 | 1 | 다음 공격 회피 | 부스터 엔진 |
| 응급 처치 | 1 | HP 6 회복 | 의무실 |
| 사기 진작 | 0 | 다음 카드 AP 비용 –1 | 선원실 |
| 생명 유지 | 패시브 | 매 턴 시작 HP +2 | 생명유지장치 |

#### 상태이상

| 상태이상 | 효과 |
|----------|------|
| 화상(Burn) | 매 턴 종료 시 n 피해 |
| 과부하(Overload) | 다음 공격 피해 2배, 이후 1턴 행동 불가 |
| EMP | 1턴 동안 카드 사용 불가 |
| 수리중(Repair) | 매 턴 n HP 회복 |

## 4. 우주선 종류

플레이 시작 시 3종의 함선 중 하나를 선택한다.

### 🛸 정찰선 (Scout)
- **그리드**: 5×5 다이아몬드형 (유효 셀 13개)
- **특징**: 공간이 좁아 장비 수가 적지만, 엔진 장비 시너지 보너스 보유
- **시작 덱**: 레이저 캐논 + 부스터 엔진
- **패시브**: 회피 카드 사용 시 AP +1 복원

### 🚀 순양함 (Cruiser)
- **그리드**: 7×7 사각형 (유효 셀 35개)
- **특징**: 균형 잡힌 공간, 모든 장비 카테고리 사용 가능
- **시작 덱**: 플라즈마 건 + 실드 제너레이터 + 선원실 A
- **패시브**: 없음 (범용)

### 🏰 전함 (Battleship)
- **그리드**: 9×7 직사각형 (유효 셀 49개)
- **특징**: 공간이 넓어 대형 장비 배치 가능, 기동성 낮음
- **시작 덱**: 레일건 + 강화 선체 + 생명유지장치
- **패시브**: 블록 카드 효과 +3

## 6. 경제 시스템 (탐험 & 거래)

### 재화
- **스크랩(Scrap)**: 전투 후 드롭, 상점 이용 시 사용
- **데이터 코어(Data Core)**: 희귀 이벤트 보상, 업그레이드에 사용

### 상점 구성
- 장비 3–5개 랜덤 진열
- 카드 업그레이드 서비스: 스크랩 소모 → 카드 강화 (피해 +3 등)
- 장비 제거 서비스: 스크랩 소모 → 덱에서 특정 카드 영구 삭제

### 장비 획득 경로
1. **전투 보상**: 전투 후 3개 중 1개 선택
2. **이벤트**: 특정 이벤트 결과로 획득
3. **상점**: 스크랩으로 구매

## 7. UI/UX 플로우

```
[메인 메뉴]
    └─ [함선 선택 화면]
           └─ [스테이지 맵 화면]
                   ├─ [전투 화면]
                   │       ├─ 함선 그리드 패널 (좌)
                   │       ├─ 전투 정보 패널 (중앙)
                   │       └─ 핸드 카드 패널 (하단)
                   ├─ [장비 배치 화면] ← 수리 기지 or 전투 후 보상
                   │       ├─ 함선 그리드 (좌)
                   │       └─ 장비 인벤토리 (우)
                   ├─ [상점 화면]
                   └─ [이벤트 화면]
```

### 장비 배치 인터페이스
- **드래그 앤 드롭** 방식으로 장비를 그리드에 배치
- 배치 가능 여부를 **초록/빨강 하이라이트**로 즉시 피드백
- 마우스 우클릭 or 회전 버튼으로 **장비 회전**
- 현재 덱 미리보기 패널: 배치 상태에 따라 실시간 업데이트

### 전투 인터페이스
- 상단: 적 HP / 의도 표시
- 중단: 내 HP / 블록 / AP 게이지
- 하단: 핸드 카드 (최대 7장), 드로우 덱 남은 수, 버린 덱 수
- 카드에 마우스 오버 시 상세 효과 툴팁 표시

## 8. 기술 스택 및 아키텍처

### 프론트엔드 (게임 클라이언트)
| 역할 | 후보 기술 | 선택 근거 |
|------|-----------|-----------|
| 게임 렌더링 | **Phaser 3** (Canvas/WebGL) | 2D 웹게임에 최적화, 그리드/스프라이트 지원 |
| UI 레이어 | HTML + CSS (or React 컴포넌트) | 카드 UI, 메뉴 등 DOM 기반 UI는 별도 레이어로 구성 |
| 상태 관리 | 순수 JS 모듈 (or Zustand) | 런 상태, 덱, 장비 배치 정보 관리 |
| 언어 | TypeScript | 타입 안전성 확보 |

### 백엔드 (선택 사항 — 싱글 플레이는 로컬 스토리지로 대체 가능)
| 역할 | 기술 |
|------|------|
| 저장소 | LocalStorage / IndexedDB (오프라인) |
| 리더보드(추후) | Node.js + Express + SQLite or Firebase |
